
# 📐 Padrões de Engenharia

> Ativo transversal de engenharia. Tudo aqui serve como **referência e ponto de partida** para qualquer ilha, qualquer microsserviço, qualquer time. Não pertence a nenhuma ilha específica.
> 

---

## Para que serve esta área

Aqui vivem os padrões, templates e decisões de engenharia que **todos os times consomem**. Quando você precisar criar um novo microsserviço, um novo componente ou seguir uma convenção do projeto, começa por aqui.

---

## O que fica aqui

| Categoria | Exemplos |
| --- | --- |
| **Boilerplates** | Estrutura base de microsserviço Clojure/Pedestal, worker Kafka, lib compartilhada |
| **Componentes reutilizáveis** | `main.clj` externalizado, interceptors base, health check padrão |
| **Convenções de código** | Naming, estrutura de namespaces, Railway Oriented, Data-Driven |
| **ADRs Globais** | Decisões arquiteturais que afetam toda a plataforma |
| **Guias de onboarding técnico** | Como configurar ambiente, REPL workflow, CI/CD |

---

## Como usar

1. Consulte a task relevante abaixo para entender o que já existe ou está planejado
2. Ao criar um novo microsserviço, use o boilerplate como base e **adapte** — nunca copie cegamente
3. Se identificar um padrão novo que todos deveriam seguir, abra uma task aqui
